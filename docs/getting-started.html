Getting Started with TooltipAI

1 System Requirements

Required

  • ✓ Windows 10 (version 1903+) or Windows 11
  • ✓ 64-bit x86-64 processor
  • ✓ 100 MB disk space
  • ✓ Internet connection (for AI features)

Recommended

  • ★ 8 GB+ RAM
  • ★ Any modern multi-core CPU
  • ★ Display scaling set to 100% or 125%

2 Installation

Download

Download the latest release from GitHub or the official website.

Download Latest Release

Extract & Run

TooltipAI is portable — no installer required. Extract the ZIP to any folder and run the executable.

# 1. Extract the ZIP to your preferred location
Expand-Archive -Path "$env:USERPROFILE\Downloads\tooltip-ai.zip" -DestinationPath "$env:USERPROFILE\TooltipAI"

# 2. Run TooltipAI
Start-Process "$env:USERPROFILE\TooltipAI\TooltipAI.exe"

Windows Defender / Antivirus

TooltipAI uses Windows UI Automation API to read screen elements. Some antivirus software may flag this behavior. This is expected — TooltipAI does not modify or interact with other applications. You can safely add it to your exclusion list.

3 First Launch

  1. a) When TooltipAI starts for the first time, you'll see a welcome screen with a quick overview of features.
  2. b) The app will request accessibility permissions. Click "Allow" when prompted — this is required for UIA element detection.
  3. c) TooltipAI will appear in the system tray (bottom-right corner). Right-click the tray icon for quick access to settings.
  4. d) Hover over any UI element in any supported app — a smart tooltip will appear with context and explanations.

4 Configuration

General Settings

Right-click the tray icon and select Settings to access configuration options.

Startup Behavior
Choose whether TooltipAI starts with Windows. Default: enabled.
Refresh Rate
Adjust tooltip refresh interval (50ms–500ms). Lower = more responsive, higher = less CPU.
Multi-Monitor
Enable or disable tooltip support across multiple displays.
Theme
Choose category color themes or enable custom styling.

AI Configuration

TooltipAI uses AI to generate contextual tooltips. Configure your AI settings:

  • ● AI Provider: Use the built-in AI or connect your own OpenAI API key for more powerful explanations.
  • ● Offline Mode: When internet is unavailable, TooltipAI falls back to local rule-based tooltips.
  • ● API Key: Enter your OpenAI API key in Settings > AI to unlock advanced context. Your key stays local and is never sent to our servers.

Account & Subscription

Sign in to your TooltipAI account to manage your subscription and sync settings.

# Via Settings > Account, or use the tray menu:
Sign In → Enter your email and password
Manage Subscription → Opens LemonSqueezy portal

5 Using TooltipAI

Basic Usage

  1. Open any supported application (Ableton, VS Code, Chrome, etc.)
  2. Hover your mouse over any UI element — a button, knob, slider, or menu item
  3. After a brief moment, a smart tooltip appears explaining what the element does
  4. Move the mouse away to dismiss the tooltip

What Tooltips Show

Element Name
The detected UI element (e.g., "Compressor > Threshold")
Category
Color-coded badge: Audio, Dev, Creative, Browser, Terminal
AI Explanation
Contextual description of what the element does
Gestures & Shortcuts
Mouse interactions and keyboard shortcuts for that element

Bookmarks & History Pro

Pro users can save tooltips to bookmarks for quick reference and browse their tooltip history. Access these from the TooltipAI panel or system tray menu.

7 Troubleshooting

Tooltips don't appear at all

1. Check that TooltipAI is running (look for the tray icon).

2. Verify accessibility permissions are granted in Windows Settings > Privacy & Security > Accessibility.

3. Ensure the target application is supported. TooltipAI works best with Win32 and UWP apps.

4. Try restarting TooltipAI and the target application.

Tooltips appear but are wrong or empty

1. If using AI tooltips, check your internet connection.

2. If using a custom OpenAI API key, verify it's valid and has quota remaining.

3. The element may not be accessible via UIA — some apps use custom rendering.

TooltipAI uses too much CPU

1. Increase the refresh rate in Settings > General (e.g., set to 200ms or 500ms).

2. Disable live visualizations if enabled.

3. TooltipAI normally uses under 2% CPU — if it's higher, try restarting.

Keyboard shortcuts don't work

1. Another application may be using the same shortcut. Check for conflicts.

2. Run TooltipAI as Administrator if targeting elevated windows.

3. Custom shortcuts can be configured in Settings > Shortcuts.
