API Reference

Backend API Documentation

Complete reference for TooltipAI's REST API endpoints.

Base URL

https://tooltip-ai.azurewebsites.net

Authentication

TooltipAI uses JWT (JSON Web Token) based authentication. After registering or logging in, you receive an access token and a refresh token. Include the access token in the Authorization header for authenticated requests.

Authorization: Bearer <your-access-token>

Access tokens expire after 1 hour. Use the refresh token endpoint to obtain a new access token without re-authenticating.

Endpoints

POST /api/auth/register

Create a new user account.

Request Body

{
    "email": "user@example.com",      // required
    "password": "securepassword",       // required, min 8 chars
    "displayName": "John Doe"          // optional
}

Response 201 Created

{
    "success": true,
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "dGhpcyBpcyBhIHJlZnJl...",
    "email": "user@example.com",
    "displayName": "John Doe"
}

Error Response 400 / 409

{
    "success": false,
    "error": "Email already registered"
}
POST /api/auth/login

Authenticate with email and password.

Request Body

{
    "email": "user@example.com",      // required
    "password": "securepassword"       // required
}

Response 200 OK

{
    "success": true,
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "dGhpcyBpcyBhIHJlZnJl...",
    "email": "user@example.com",
    "displayName": "John Doe"
}

Error Response 401

{
    "success": false,
    "error": "Invalid email or password"
}
POST /api/auth/refresh

Refresh an expired access token using a refresh token.

Request Body

{
    "refreshToken": "dGhpcyBpcyBhIHJlZnJl..."  // required
}

Response 200 OK

{
    "success": true,
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "bmV3IHJlZnJlc2ggdG9r..."
}

Error Response 401

{
    "success": false,
    "error": "Invalid or expired refresh token"
}
GET /api/auth/profile Auth Required

Get the current user's profile information.

Headers

Authorization: Bearer <access-token>

Response 200 OK

{
    "success": true,
    "user": {
        "id": "usr_abc123",
        "email": "user@example.com",
        "displayName": "John Doe",
        "plan": "pro",
        "createdAt": "2026-01-15T08:30:00Z"
    }
}
GET /api/auth/check Auth Required

Check if the current access token is valid. Useful for validating session state.

Headers

Authorization: Bearer <access-token>

Response 200 OK

{
    "success": true,
    "authenticated": true,
    "userId": "usr_abc123"
}

Error Response 401

{
    "success": false,
    "authenticated": false
}
POST /api/webhooks/lemonsqueezy Internal / Signature Verified

Webhook endpoint for LemonSqueezy subscription events. Handles payment confirmations, subscription updates, and cancellations. Requests must include a valid LemonSqueezy signature header.

Headers

X-Signature: <lemonsqueezy-signature>
Content-Type: application/json

Events Handled

● subscription_created — Activates Pro/Business features
● subscription_updated — Handles plan changes
● subscription_cancelled — Reverts to free tier at period end
● order_created — Records payment
GET /health

Health check endpoint. Returns server status, connected clients, and WebSocket port.

Response 200 OK

{
    "status": "ok",
    "clients": 1,
    "ws_port": 8765,
    "name": "tooltip-ai-backend",
    "version": "1.0.0"
}
GET /ready

Readiness probe. Returns 200 when the server is ready to accept traffic (used by container orchestrators).

Response 200 OK

{
    "ready": true
}

Rate Limits

API requests are rate-limited per account to ensure fair usage:

● Authentication endpoints: 10 requests per minute per IP
● Profile/Check endpoints: 60 requests per minute per token
● Health/Ready: No limit

Error Format

All error responses follow a consistent format:

{
    "success": false,
    "error": "Human-readable error message"
}
200 Success
201 Created
400 Bad Request — missing or invalid fields
401 Unauthorized — invalid or expired token
409 Conflict — resource already exists (e.g., email taken)
500 Internal Server Error
